Building Express applications made easy!


Banga is a cli generator for building ExpressJS applications with speed and efficiency.

GitHub NPM | Author

Get started

Banga can be installed globally via npm.

$ npm install bangajs -g

Usage

After installation you can invoke the tool on the command line via the banga command.


To confirm successful installation run the command below to check your installed version.

$ banga version

BangaJS v0.0.0

Now let's create a new Express app named hello-app. It would be created in a new folder called hello-app in the current working directory.

$ banga new hello-app
...

Then navigate into the newly created project and start up the server. By default the server should be running on http://localhost:2020/

$ cd hello-app
$ npm start

Help

You can get help tips for commands usage.

$ banga --help
$ banga <command> --help

Commands

Here's a list of available commands

command Alias Description
docs d Opens Banga docmentation in a browser
generate g Generates and/or modifies files.
help h Lists available commands and their short descriptions.
new n Creates a new workspace and an initial Express app.
version v Displays Banga version.

docs

Opens Banga docmentation in a browser

$ banga docs
$ banga d

generate

Generates and/or modifies files

$ banga generate <type> <name> [options]
$ banga g <type> <name> [options]

Example:
$ banaga generate controller user --no-crud
$ banaga g asset user
$ banaga generate service user
$ banaga g r user
$ banaga generate m user

Types

Type Alias Description
asset a Create controller, model, route and service files.
controller c Create controller file
model m Create model file
route r Create route file
service s Create service file

Options

Option Description
--no-crud Do not include CRUD methods
--force Replace file if it already exists
--root Create file at the root of the current working directory

help

Lists available commands and their short descriptions.

$ banga help
$ banga h

new

Creates a new workspace and an initial Express app.

$ banga new <name> [options]
$ banga n <name> [options]

Example:
$ banaga new hello-app --no-auth
$ banaga new myap
$ banaga n

Options Description

Displays Banga version..

$ banga version
$ banga v
